Welcome everybody to my fresh webpage. Every now and then I’ll post short little programming applications that I think might be helpful for people out there. Most if this will be related to NetLogo and R and the combination of both tools.

In this first post, I will show you how you can use R colorramps, such as the wonderful viridis colorramp, directly in your NetLogo models.

Colormaps

The R package Viridis provides color palettes that are optimized to represent data. The colorramps are easier to read by those with colorblindness and can be easily distinguished when printed in grey scale.

I use viridis colorramps for many data visualization purposes every day. However, I also work quite a lot with NetLogo models and was always missing the possibilty to use nice color palettes for coloring the cells or turtles according to certain properties of these agents.

So I decided to implement a neat little function in NetLogo that can be copied to any of your NetLogo models. It will allow you to use the viridis color palettes and many other palettes directly within NetLogo.

Extract color values from the viridis palettes

In order to implement the viridis colorramp into NetLogo, we first need to extract the RGB color values from the viridis palette. In the following code snippet, we use the colormap package to create a palette with RGB values. Afterwards, we write the palette values to a *.txt file in a “NetLogo-friendly” format.

library(viridis)
library(colormap)
# Define length of viridis palette -> n > 300 doubles some colors
n <- 300
## Create a viridis palette rgb dataframe
virpal <- colormap(colormap = colormaps$inferno, format = 'rgb', nshades = n)
## Remove alpha channel
virpal <- virpal[,-4]

# Write as file
sink("viridis.txt")
cat("[")
for (i in 1:nrow(virpal))
{
  cat("[")
  cat(paste0(virpal[i,]))
  cat("]")
}
cat("]")
sink()

The colormap package supports many different colorramps, including additional viridis palettes, such as the magma or plasma palette. A complete list can be found here. In order to create output files for these colorramps, you only need to adjust the colormaps$viridis parameter within the colormap function call. For example, colormap(colormap = colormaps$magma, format = 'rgb', nshades = n) will report n values of the magma palette.

Second, we need a function that scales our color palette to the variable values of agents. This function works similar to the scale-color primitive of NetLogo. It will take 4 arguments. First, a string to select the palette that you want to use for coloring. This allows you to define multiple color palettes with different reporter functions (see examples in NetLogo example model at the buttom of this post). Second, it vill take the variable value of the current agent. The remaining two values define the variable range. The function will then scale the variable range onto the range of the selected color palette and selects the item that corresponds to the variable value of the actual agent.

to-report scale-colormap [palette var minvar maxvar]
  ;; Rescale the variable to 0..300
  ;;The transformation works as follows:
  ; NewValue = (((OldValue - OldMin) * (NewMax - NewMin)) / (OldMax - OldMin)) + NewMin
  report item floor(((var - minvar) * (length(run-result (word "palette-" palette)) - 1)) / (maxvar - minvar)) run-result (word "palette-" palette)
end

Third, we may also want to define a function that allows us to pick a specific color from a palette. This function also checks if the requested index value is available within the selected palette.

to-report colormap [palette palette_index]
  ;; Scale a variable to the length of the palette:
  let pal_viridis palette-viridis

  ;; Check vir_i:
  ifelse (palette_index >= length(run-result (word "palette-" palette)))
  [
    print (word "viridis color index invalid. Index maximum = " (length(run-result (word "palette-" palette)) - 1))
    report item 0 run-result (word "palette-" palette)
  ]
  [
    report item palette_index run-result (word "palette-" palette)
  ]
end

These functions can now be stored as a separated *.nls file. This file can then be included in any NetLogo model via the __includes[] primitive.

Complete NetLogo example including four palettes

You can download a NetLogo example model here: colormap.nlogo; colormap.nls

The colormap.nls file includes the described functions above and can be included to any NetLogo model. The colormap.nlogo model shows some examples of these functions in action. Please download both and put the in the same folder and open the colormap.nlogo file to run the examples. Of course you can edit the colormap.nls file and add more palette reporters of any palettes you wish to use.
